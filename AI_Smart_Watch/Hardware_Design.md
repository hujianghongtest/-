# AI 智能助手手表 — 硬件设计方案

## 项目概述

基于 ESP32-S3-Zero 开发板设计的穿戴式 AI 智能助手手表。具备语音对话、屏幕显示、环境感知、情感交互等功能。

### 核心功能
- **AI 语音对话**：通过 WiFi 连接云端 AI（如 OpenAI / 百度文心 / 阿里通义千问），实现自然语言对话
- **情感关怀**：基于时间、环境数据、交互频率，主动问候并给出生活建议
- **屏幕显示**：圆形/方形 TFT 屏幕，显示时间、温度、天气、AI 对话内容
- **环境感知**：温度、湿度、气压实时采集
- **触控/按键交互**：实体按键 + 可选触摸屏

**引脚使用率: 13/13 (100%)**

### 3.3 引脚优化技术

#### TFT_RST 省略方案
- 显示屏 RST 引脚通过 10kΩ 上拉至 3.3V, 并联 0.1μF 至 GND
- 上电时 RC 电路 (τ=1ms) 自动产生复位脉冲
- TFT_eSPI 设置 `#define TFT_RST -1` 使用 `writecommand(TFT_SWRST)` 软件复位
- GC9A01 和 ST7789 均支持软件复位命令 (0x01)

#### 3按键 ADC 电阻分压方案
```
3.3V ─┬─ R1=10kΩ ─┬─ BTN_OK   ──── GND   → ADC≈0.00V
      ├─ R2=10kΩ ─┬─ BTN_DOWN ──── GND   → ADC≈1.65V
      ├─ R3=10kΩ ─┬─ BTN_UP    ──── GND   → ADC≈2.20V
      │
      └─ R4=100kΩ ── GND (分压下臂)
      
检测点(接GPIO1) → 无按键=3.3V, 各键电压间隔≥0.55V
```
按键识别阈值: OK<0.8V, DOWN 1.1~2.0V, UP>2.0V, 无按键>2.8V

#### I2S 全双工共享时钟
```
ESP32-S3 (I2S Master, 全双工模式)
    BCLK(GPIO5) ──┬── INMP441 SCK (输入)
                   └── MAX98357 BCLK (输出)
    WS(GPIO6)   ──┬── INMP441 WS
                   └── MAX98357 LRC
    SD_IN(GPIO7) ←── INMP441 SD (录音数据)
    DOUT(GPIO8) ───→ MAX98357 DIN (播放数据)
    
模式: I2S_MODE_MASTER | I2S_MODE_RX | I2S_MODE_TX
采样率: 16kHz / 16bit / 单声道
```

## 四、子系统详细设计

### 4.1 电源管理

```
[USB 5V] → [TP4056] → [LiPo 3.7V 500mAh]
                 │
                 ├→ DW01 保护板 (TP4056 模块自带)
                 │
                 └→ [3.3V LDO] → ESP32-S3 + 外设 (≤500mA)
                        │
                        └→ 100μF 钽电容滤波
```

- **充电**：TP4056 通过 USB Type-C 充电，最大 1A
- **放电保护**：TP4056 模块自带 DW01 锂电保护芯片
- **续航估算**：500mAh / ~80mA (平均功耗) ≈ 6小时
- **待机优化**：ESP32-S3 深度睡眠模式 ~5μA，配合 RTC 唤醒

### 4.2 音频子系统

```
[INMP441 Mic] → I2S → [ESP32-S3] → I2S → [MAX98357] → [Speaker]
                      │
                      ├→ 音频采集: 16kHz, 16bit, 单声道
                      ├→ 回声消除: 软件 AEC (ESP-ADF)
                      ├→ 语音识别: 云端 ASR (HTTP/WebSocket)
                      ├→ AI 回复: 云端 LLM API
                      └→ TTS 合成: 云端 TTS 或本地 ESP-TTS
```

- **麦克风**：INMP441 数字 MEMS，高灵敏度 (-26dBFS)，L/R 接 GND (左通道)
- **功放**：MAX98357，I2S 输入，3W @ 4Ω，SD 接 3.3V (常开)
- **语音链路**：本地采集 → 云端 ASR → LLM 推理 → 云端 TTS → 本地播放
- **时钟共享**：BCLK 和 WS 由 ESP32-S3 作为 I2S Master 统一提供

### 4.3 显示子系统

```cpp
// 显示初始化参数 (GC9A01 / ST7789)
SPI Freq:     40 MHz (标准模式)
Resolution:   240 × 240
Color Depth:  16-bit (RGB565)
Orientation:  根据佩戴方向软件可调
RST:          软件复位 (TFT_RST = -1), 硬件RC上电复位
BL:           GPIO9 LEDC PWM 调光
```

- **UI 框架**：LVGL (Light and Versatile Graphics Library)
- **界面元素**：表盘时钟、温度/天气卡片、AI 对话气泡、设置菜单
- **背光**：PWM 控制，亮度 0~100%

## 七、注意事项与建议

### 7.1 电源注意事项
- MAX98357 峰值电流可达 1A，需在 VIN 端并联 100μF 电容
- ESP32-S3 WiFi 发射时功耗约 300-500mA，需确保 3.3V LDO 输出能力 ≥ 500mA
- 电池端建议增加 10μF + 0.1μF 去耦电容

### 7.2 信号完整性
- I2S 时钟线 (BCLK/WS) 尽量短，远离 WiFi 天线区域
- I2C 总线长度 < 15cm，否则降低速率
- SPI 时钟 40MHz 时，连线 < 10cm

### 7.3 GPIO3 安全性
- GPIO3 是 ESP32-S3 的 JTAG_TCK strapping 引脚
- 出厂默认 eFuse 状态下，JTAG 固定走 USB，strapping 值被忽略
- I2C 的 4.7kΩ 上拉电阻使 GPIO3 在启动时保持 HIGH (对应 USB JTAG 模式)
- 未烧 eFuse 时完全安全；如需烧 JTAG eFuse，仍需保持 HIGH

### 7.4 天线注意事项
- ESP32-S3-Zero 板载陶瓷天线
- 手表外壳需为非金属材质（塑料/树脂/3D打印 PLA）
- 金属表带可能影响 WiFi 信号，建议皮/硅胶/尼龙表带

### 7.5 散热
- 持续 WiFi + 音频播放时，ESP32-S3 温度可达 50°C+
- 手表底部与皮肤接触处需做隔热处理
- 建议在外壳上开散热孔

### 7.6 软件架构建议
```
firmware/
├── src/
│   ├── main.cpp           # 入口，初始化 + 主循环
│   ├── display_task.cpp   # 显示任务 (LVGL)

│   ├── audio_task.cpp     # 音频采集与播放
```

## 八、实物组装步骤

1. **PCB/洞洞板布局**：按照连接表规划元器件位置，注意模拟/数字区域分离
2. **焊接电源部分**：TP4056 → 电池插座 → 3.3V 稳压 → 去耦电容
3. **焊接 I2C 总线**：BME280 + DS3231 + 上拉电阻 (GPIO3=SDA, GPIO4=SCL)
4. **焊接 SPI 显示屏**：GPIO9-13 (注意: RST 接 RC 电路, 不接 GPIO)
5. **焊接音频部分**：麦克风远离扬声器，避免啸叫；BCLK(GPIO5)和WS(GPIO6)共享连接
6. **焊接按键**：3按键 + 4电阻梯形分压网络 → GPIO1
7. **焊接电池分压**：100kΩ+100kΩ 分压 → GPIO2
8. **上电测试**：先不焊接电池，USB 供电测试所有功能
9. **组装外壳**：3D 打印外壳，安装表带
